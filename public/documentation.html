MyFlix

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Businnes Logic URL HTTP Method Request body data format Response body data format
Get a list of movies /movies GET none A JSON object holding data about all movies
Get data about a single movie by title /movies/[title] GET none A JSON object holding data about a single movie, containing description, genre, director, image URL, whether it’s featured or not. Example: { "genre": { "name": "Historical drama", "description": "The historical drama..." }, "director": { "name": "Christopher Nolan", "bio": "Christopher Nolan is...", "birth": "1970", "death": "" }, "actors": [], "_id": "64faff05e32932f5a23ea847", "title": "Dunkirk", "description": "Allied soldiers from Belgium, the British Commonwealth...", "image": "https://www.google.com/...", "featured": true }
Get data about genre /movies/genre/[genreName] GET none A JSON object holding data about a genre of each movie. Example: "genre": { "name": "drama", "description": "On top of all of that, the movie is a..." }
Get data about director /movies/director/[directorName] GET none A JSON object holding data about a director of each movie. Example: { "name": "Antoine Fuqua", "bio": "Antoine Fuqua is ...", "birth": "1965", "death": "" }
Get all users /users GET none A JSON object holding data about all movies
Create a new user /users POST A JSON object holding data about a user to add, structured like: { "name": "Anjelina Jolie", "password": "rrrrr", "email": "anjelka@gmail.com", "birthday": "01/01/1999" } A JSON object holding data about a user that was added, inluding an ID: { "name": "Anjelina Jolie", "password": "rrrrr", "email": "anjelka@gmail.com", "birthday": "1998-12-31T23:00:00.000Z", "favoriteMovies": [], "_id": "6500966796de468c0e3a64a8", "__v": 0 }
Remove a user by Name /users/[name] DELETE none A text message indicating whether a user was successfully removed.
Update a user's info (password) /users/[name] PUT A JSON object holding user's data that must be updated: { "password": "12345" } A JSON object holding data about a user that was updated: { "_id": "6500966796de468c0e3a64a8", "name": "Anjelina Jolie", "password": "12345", "email": "anjelka@gmail.com", "birthday": "1998-12-31T23:00:00.000Z", "favoriteMovies": [], "__v": 0 }
Add a movie to user favorite list /users/:[username]/movies/:[movieID] POST none A JSON object holding user's data: { "_id": "6500966796de468c0e3a64a8", "name": "Anjelina Jolie", "password": "12345", "email": "anjelka@gmail.com", "birthday": "1998-12-31T23:00:00.000Z", "favoriteMovies": [ "64fb03bae32932f5a23ea84b" ], "__v": 0 }
Remove a movie from user favorite list /users/:[username]/movies/:[movieID] DELETE none A JSON object holding user's data: { "_id": "6500966796de468c0e3a64a8", "name": "Anjelina Jolie", "password": "12345", "email": "anjelka@gmail.com", "birthday": "1998-12-31T23:00:00.000Z", "favoriteMovies": [], "__v": 0 }
